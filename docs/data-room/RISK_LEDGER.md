# SMIRK / CaseFlow Risk Ledger

Date: 2026-06-04

| ID | Severity | Area | Risk | Evidence | Business Impact | Mitigation |
| --- | --- | --- | --- | --- | --- | --- |
| R-001 | High | Multi-tenancy | Tenant isolation is application-enforced, not DB-enforced. | `getWorkspaceId` trusts `X-Workspace-Id` and defaults to `1`; comments say every query must use it (`server.ts:374` to `server.ts:381`). | Cross-tenant data exposure if a route misses `workspace_id`. | Add RLS or scoped query layer; deny unscoped queries in tests; remove fail-open defaults for SaaS mode. |
| R-002 | High | Multi-tenancy | Legacy tables default `workspace_id` to `1`. | Workspace columns added with `DEFAULT 1` across core tables (`src/db.ts:458` to `src/db.ts:479`). | Old single-tenant behavior can leak into new tenant flows. | Backfill intentionally, then require explicit workspace IDs and add FK constraints to `workspaces`. |
| R-003 | High | Telecom cost | Twilio subaccount/number lifecycle is incomplete. | Provisioning creates subaccounts and purchases numbers (`src/twilio-provisioning.ts:61` to `src/twilio-provisioning.ts:126`); `deleteWorkspace` only deletes DB row (`src/saas.ts:277` to `src/saas.ts:279`). | Orphaned paid numbers/subaccounts can burn cash. | Add cancellation/deletion workflow that releases numbers, closes subaccounts, and writes release status before DB deletion. |
| R-004 | High | Telecom routing | Phone-number workspace lookup can fail open to workspace `1`. | Lookup catches errors and returns `1` (`server.ts:432` to `server.ts:434`). | Wrong workspace could answer if mapping query fails. | In SaaS mode, fail closed for unknown/failed phone mapping. |
| R-005 | High | AI key security | Workspace AI keys appear stored as plaintext columns. | `openrouter_api_key`, `elevenlabs_api_key`, `gemini_api_key` are workspace columns (`src/saas.ts:106` to `src/saas.ts:108`); no encryption wrapper found equivalent to Twilio token encryption. | Increased breach impact and diligence concern. | Encrypt workspace secrets at rest; add key rotation audit log; mask values in all APIs. |
| R-006 | Medium | Key rotation | Key cache exists, but no full rotation lifecycle is evident. | 5-minute cache and invalidate function exist (`src/workspace-ai-keys.ts:35`, `src/workspace-ai-keys.ts:108` to `src/workspace-ai-keys.ts:112`). | Rotated keys may take time to propagate; limited auditability. | Add audited rotate/test/revoke endpoint and automatic cache invalidation on settings update. |
| R-007 | High | Hallucination control | Post-call evaluator is non-blocking and runs after persistence. | Evaluator is called after `persistCallSummary` and failures are non-critical (`src/intelligence.ts:640` to `src/intelligence.ts:697`). | Bad extraction can be written before review. | Add pre-write verifier for extracted entities/tasks with transcript evidence requirements. |
| R-008 | Medium | Provenance | Tasks do not consistently store transcript snippet evidence. | Tasks table links `call_sid` but has no transcript snippet column (`src/db.ts:267` to `src/db.ts:280`); `createLead` task writes notes without snippet (`src/tools.ts:75` to `src/tools.ts:79`). | Harder to prove source-to-task lineage in regulated sales/legal contexts. | Add `source_message_ids`, `transcript_snippet`, `extraction_confidence`, and `verified_by` columns to tasks. |
| R-009 | Medium | Integration tenancy | Plugin tools are schema-scoped by workspace but loaded without workspace filtering. | `getPluginTools` selects enabled tools without `workspace_id` predicate (`src/plugin-tools.ts:57` to `src/plugin-tools.ts:64`). | One tenant’s custom tool could become visible/callable in another tenant. | Pass workspace ID into plugin/MCP loaders and enforce workspace predicates. |
| R-010 | Medium | CRM readiness | Salesforce integration uses simplified Account mapping. | `AccountId = contact.company` (`src/crm.ts:277` to `src/crm.ts:283`). | Enterprise CRM sync may fail or dirty data. | Build CRM-specific mapping config, object discovery, sandbox tests, and per-tenant credentials. |
| R-011 | Medium | Unit economics | Input/output token accounting is not first-class. | Ops monitor sums optional `tokensUsed`; no input/output split (`server.ts:7196` to `server.ts:7204`). | Cannot prove gross margin by customer or intake type. | Persist provider, model, prompt_tokens, completion_tokens, cached_tokens, cost_usd per AI call. |
| R-012 | Medium | Usage limits | Usage-limit failure allows calls. | Catch block logs warning and allows call (`server.ts:2463` to `server.ts:2476`). | Cost exposure during DB/provider outage. | For paid SaaS, define policy: fail closed after grace threshold or route to voicemail-only. |
| R-013 | Medium | Runtime scaling | Process memory is used for timers/audio/response state. | Call timeout timers and TTS audio store are in-process (`server.ts:2561` to `server.ts:2573`, `server.ts:1452` to `server.ts:1546`). | Multi-instance deployments can lose state or create inconsistent behavior. | Move call orchestration state to Redis/queue/object storage; use sticky or event-driven processing. |
| R-014 | Medium | Tool safety | AI can call arbitrary operator-defined HTTP tools. | Plugin tools include custom URL, method, headers, params (`src/plugin-tools.ts:30` to `src/plugin-tools.ts:43`) and execute via `fetch` (`src/plugin-tools.ts:160` to `src/plugin-tools.ts:231`). | SSRF/data exfiltration risk if operators misconfigure or accounts are compromised. | Add allowlists, egress policy, secret vault, tool permission scopes, and audit review. |
| R-015 | Low | Schema management | No Drizzle schema/migration discipline found. | DDL is handwritten/idempotent in `src/db.ts` and `src/saas.ts`. | Diligence friction and migration drift risk. | Introduce formal migration tooling with checked-in migrations and schema diff CI. |
| R-016 | Medium | OpenClaw/Knot claims | No `Knot` framework reference was found in this checkout. | Runtime comments and code refer to OpenClaw/OpenRouter/Gemini, not Knot (`server.ts:1673` to `server.ts:1732`). | Investor materials may overclaim architecture. | Either include the Knot repo in diligence or remove/rephrase the claim. |
